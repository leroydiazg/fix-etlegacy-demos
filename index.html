filestatusmod_version
Root cause: CG_ParseDemoVersion() in ET:Legacy's cgame calls strtok(versionStr, ".") three times expecting major.minor.patch. A missing part makes the third call return NULL, and Q_atoi(NULL) → strtol(NULL, ...) is undefined behaviour → crash. CG_ParseDemoVersion() re-runs every time a new gamestate is processed — not just at the start — because servers commonly resend a full gamestate mid-demo (e.g. via map_restart when warmup ends). This tool decodes the demo's adaptive-Huffman-coded messages (a faithful JS port of ET:Legacy's own huffman.c / msg.c), scans every frame of the file for svc_gamestate messages, and rewrites mod_version in all of them — everything else is copied through unchanged. Folder mode scans recursively for any .dm_<n> file.

Source: github.com/etlegacy/etlegacy (GPLv3)
